#include<queue>

// 默认优先从小到大输出

std::priority_queue<Type,Container,Functional>

//期中type是数据类型，第二个是保存数据的容器，第三个是元素比较函数，默认从大到小operator<

//容器默认为vectord

//如果优先输出小数据则使用，

priority_queue<int,vector<int>,greater<int>> p;

// less则是大顶堆，降序
//如果使用自定义的节点

struct node
{
    int x;
    int y;
    node(int a,int b):x(a),y(b){}
};

struct cmp
{
    bool operator()(node a,node b)
    {
        if(a.x==b.x) return a.y>b.y;
        else
            return a.x>b.x;//从小到大
    }
};
priority_queue<node,vector<node>,cmp> p;

这里很容易混淆的一点是priority_queue是默认大顶堆的，所以a.x<b.x是降序，a.x>b.x是升序。

但是sort是默认升序的，所以a.x<b.x是升序，a.x>b.x是降序。


优先队列自定义排序方式

重载operater<或operator>
创建大顶堆时重载operator<，对应less方法，元素降序
bool operator<(node a,node b)
{
    if(a.x==b.x) return a.y>b.y;
    else
        return a.x<b.x;//从小到大
}
创建小顶堆时重载operator>，对应greater方法，元素升序


声明比较类cmp
struct cmp
{
    bool operator()(node a,node b)
    {
        if(a.x==b.x) return a.y>b.y;
        else
            return a.x>b.x;//从小到大
    }
};


但是sort还多一个自定义比较函数cmp

bool cmp(node a,node b)
{
    return a.x<b.x;
}

sort(vec.begin(),vec.end(),cmp())

两个自定义函数对比
priority_queue有两种自定义方式：重载操作符和声明比较类；sort相比于前者增加一个定义比较函数；

默认排序方式不同：priority_queue默认为大顶堆，降序；sort默认升序

自定义运算方式不同：比如"a < b"在priority_queue中代表降序，而在sort中代表升序

调用方式不同：定义比较类时，priority_queue调用过程中不需要"()"–>cmp，而sort无论调用自定义比较函数还是比较类都需要加上"()"–>cmp()
