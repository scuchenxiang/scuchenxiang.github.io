即使将 N 除以 25 之后，仍然没法在短时间内得到答案，因此我们需要尝试一些别的思路。可以发现，分配操作有 (4, 0)，(3, 1)，(2, 2) 和 (1, 3) 四种，那么在一次分配中，汤 A 平均会少 (4 + 3 + 2 + 1) / 4 = 2.5 份，汤 B 平均会少 (0 + 1 + 2 + 3) / 4 = 1.5 份。因此在 N 非常大的时候，A 会有很大的概率比 B 先分配完，所有概率应该非常接近 1。事实上，当 N >= 500 25 时，所求概率已经大于 0.999999 了（可以通过上面的动态规划方法求出），它和 1 的误差（无论是绝对误差还是相对误差）都小于 10^-6。因此在 N >= 500 25 时，我们只需要输出 1 作为答案即可。在其它的情况下，我们使用动态规划计算出答案。

double soupServings(int N) 
{
    N=N/25+(N%25>0?1:0);
    if(N>=500*25)
        return double(1);
    vector<vector<double> > res(N+1,vector<double>(N+1,0));
    for(int i=1;i<=N;i++)
    {
        res[0][i]=1.0;
    }

    res[0][0]=0.5;
    for(int i=1;i<=N;i++)
    {
        for(int j=1;j<=N;j++)
        {

            double tmp=res[(i-4)<0?0:(i-4)][j]+res[(i-3)<0?0:(i-3)][(j-1)<0?0:(j-1)];
            double tmp1=res[(i-2)<0?0:(i-2)][(j-2)<0?0:(j-2)]+res[(i-1)<0?0:(i-1)][(j-3)<0?0:(j-3)];
            res[i][j]=0.25*(tmp+tmp1);
        }
    }
    return res[N][N];
}
